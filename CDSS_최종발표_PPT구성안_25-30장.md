### 슬라이드 4: 시스템 범위 및 사용자 역할
**제목**: 시스템 범위 및 사용자 역할

**두 가지 플랫폼**:

1. **병원 홈페이지 (Public) - 환자 전용**
   - 환자 회원가입/로그인
   - 본인 정보 조회
   - 진료 기록 조회
   - 의료 이미지 조회
   - 예약 관리
   - Flutter 모바일 앱 (피부암 1차 스크리닝)

2. **의료진 플랫폼 (CDSS) - 의료진 전용**
   - 통합 대시보드
   - AI 분석 시스템
   - OCS/RIS/LIS 통합
   - PACS 연동
   - 의사 인터페이스 (외과, 호흡기내과 등)
   - 원무과 인터페이스
   - 방사선과 인터페이스
   - 영상의학과 인터페이스
   - 검사실 인터페이스

**6가지 사용자 역할**:
1. **환자 (Patient)**: 본인 의료 정보 조회 및 예약 관리, 모바일 앱을 통한 건강 관리
2. **원무과 (Admin)**: 예약 관리, 수납 처리, 환자 정보 관리
3. **방사선과 (Radiology)**: 의료 영상 촬영, 영상 데이터 관리, PACS 시스템 연동
4. **영상의학과 (Imaging)**: AI 분석 결과 확인, 판독서 작성, 영상 판독 및 진단 지원
5. **검사실 (Lab)**: 검사 수행 및 결과 입력, 검사 데이터 관리 및 전달, AI 예측 활용
6. **의사 (Doctor)**: 진단, 처방, 판독서 확인, 환자 정보 조회, AI 분석 결과 활용

**전체 워크플로우**:

**1. 환자 (Patient)**:
- **Mobile/Web**: 예약 신청 및 문진표 작성
- **예약 정보**: 날짜, 시간, 검사 유형 선택
- **문진표**: 증상, 과거력, 가족력 입력

**2. 원무과 (Admin)**:
- **접수 확인**: 환자 예약 정보 검토
- **대기열 등록**: 예약을 시스템에 등록
- **환자 정보 확인**: 기본 정보 및 문진표 확인

**3. 의료진 - 방사선과/영상의학과 (Radiology/Imaging)**:
- **검사 수행**: 맘모그래피, MRI, CT 등 영상 촬영
- **PACS 업로드**: Orthanc에 DICOM 형식으로 영상 저장
- **AI 분석 자동 트리거**: 영상 업로드 시 자동으로 AI 분석 요청
  - 맘모그래피 → ResNet50 (Mosec 5007)
  - MRI → SwinUNETR (Mosec 5006)
  - 병리 이미지 → CLAM/nnUNet (교육원 PC 워커)

**4. 의료진 - 검사실 (Lab)**:
- **검사 수행**: 혈액검사, RNA 검사 등
- **결과 입력**: 검사 결과를 시스템에 입력
- **AI 예측**: 폐암/pCR 예측 (Flask ML Service 5002)

**5. 의사 (Doctor)**:
- **진료실 접근**: 환자 정보 및 검사 결과 확인
- **AI 결과 확인**: 
  - 맘모그래피 AI 분석 결과 (히트맵 포함)
  - MRI 세그멘테이션 결과
  - 병리 이미지 분석 결과
  - 폐암/pCR 예측 결과
- **처방 작성 (OCS)**: 
  - 처방전 생성
  - 약물 상호작용 검사 (FastAPI 8002)
  - 알레르기 검사
  - 주문 전달 (약국, 검사실, 영상의학과)

**6. 약국/검사실/영상의학과**:
- **주문 수신**: OCS를 통해 주문 확인
- **처리 완료**: 작업 완료 후 상태 업데이트
- **알림 전송**: 의사에게 완료 알림

**역할별 상호작용 다이어그램**:
```
[환자] ──예약──> [원무과] ──접수──> [의료진]
                                              │
                                              ├─> [방사선과] ──영상 업로드──> [AI 분석]
                                              │                                    │
                                              └─> [검사실] ──검사 결과──> [AI 예측] │
                                                                                    │
[의사] <──AI 결과── [Django] <──분석 결과── [AI 서버들] ────────────────────────────┘
    │
    └──처방(OCS)──> [약국/검사실/영상의학과]
```

**시각 자료**:
- 플랫폼 구조도
- 역할별 아이콘 및 설명
- 6가지 역할 워크플로우 다이어그램

### 슬라이드 7: 데이터베이스 설계
**제목**: 데이터베이스 설계

**MySQL - 총 25개 테이블, 5개 카테고리**:

1. **사용자/인증 (User/Authentication)** - 2개
   - `auth_user` (의료진 계정)
   - `patient_user` (환자 계정)
   - **기능**: 의료진 및 환자 계정 관리, 역할 기반 접근 제어 (RBAC)

2. **환자 관리 (Patient Management)** - 4개
   - `patients_patient` (환자 기본 정보)
   - `patients_appointment` (예약 정보)
   - `patients_medicalrecord` (진료 기록)
   - 기타 환자 관련 테이블
   - **기능**: 환자 기본 정보, 예약 관리, 진료 기록 저장

3. **OCS (처방전달시스템)** - 8개
   - `ocs_order` (주문 정보)
   - `ocs_imaginganalysisresult` (맘모그래피 분석 결과 포함)
   - `ocs_pathologyanalysisresult` (병리 분석 결과)
   - `ocs_labtestresult` (검사 결과)
   - `ocs_orderstatushistory` (주문 상태 이력)
   - `ocs_druginteractioncheck` (약물 상호작용 검사)
   - `ocs_allergycheck` (알레르기 검사)
   - `ocs_notification` (알림)

### 슬라이드 16: 유방암 모델 분석 - 맘모그래피
**제목**: 맘모그래피 AI 분석 (Mammography)

**데이터셋 (VINDR-MAMMO)**:
- **연구 대상**: 유방암 조기 진단을 위한 맘모그래피 AI 판독 지원
- **크기**: 총 20,000장
- **그룹 구성**:
  - Normal(정상): 18,247장
  - Mass(종괴): 1,113장
- **형식**: DICOM
- **뷰**: L-CC, R-CC, L-MLO, R-MLO (좌/우 두미-꼬리, 내측-외측 경사)

**전처리**:
- 변환: DICOM → NumPy
- 반전: Otsu 이진화
- 배경 제거: 윤곽선 검출
- 영역 검출: 바운딩 박스 크롭
- 리사이즈: 종횡비 유지 512×512 리사이즈

**증강**:
- 전처리 단계에서 수평/수직 플립
- ±50° 회전 적용 후 저장

**모델**:
- Architecture: ResNet50 (ImageNet pretrained)
- Optimizer: Adam (lr = 0.001)
- Loss Function: CrossEntropyLoss
- Batch Size: 16
- Epochs: 100

**성능 (Mass)**:
- Precision: 93.04%
- Recall: 89.59%
- F1-Score: 91.28%
- AUC: 93.23%

**시스템 통합**:
- 서비스: Mosec (Port 5007, GCP 서버)
- 통합 방식: Django → Orthanc → Mosec → 결과 저장
- 자동화: 영상 업로드 시 자동 분석 트리거
- 결과 활용: 의사가 진료실에서 AI 결과 확인 후 판독

### 슬라이드 20: 핵심 시스템 (OCS, RIS, LIS)
**제목**: 핵심 시스템 (OCS, RIS, LIS)

**OCS (Order Communication System) - 처방전달시스템**:
- **주문 관리**: 처방, 검사, 영상 주문 통합 관리
- **주문 상태 추적**: 주문 생성 → 승인 → 수행 → 완료
- **약물 상호작용 검사**: FastAPI를 통한 약물 검색 및 상호작용 검사
- **알레르기 검사**: 환자 알레르기 정보 기반 검사
- **AI 분석 결과 저장**: 맘모그래피, 병리 분석 결과 저장

**RIS (Radiology Information System) - 영상의학 정보시스템**:
- **검사 스케줄링**: 영상 검사 일정 관리
- **검사 결과 관리**: 판독서 작성 및 관리
- **PACS 연동**: Orthanc과의 DICOM 데이터 연동

**LIS (Laboratory Information System) - 검사실 정보시스템**:
- **검사 결과 입력**: 혈액검사, RNA 검사 등 검사 결과 입력
- **검사 데이터 관리**: 검사 결과 저장 및 전달
- **AI 예측 연동**: 폐암/pCR 예측 (Flask ML Service)

**시스템 통합**:
- OCS, RIS, LIS 완전 통합
- 실시간 데이터 동기화
- 의료진 간 협업 지원

**시각 자료**:
- OCS/RIS/LIS 통합 구조도
- 주문 흐름도

### 슬라이드 8: 데이터베이스 설계
**제목**: 데이터베이스 설계

**주요 테이블**:
1. **환자 관리**
   - `PatientUser`: 환자 계정 정보
   - `Patient`: 환자 기본 정보
   - `Appointment`: 예약 정보

2. **의료진 관리**
   - `User`: 의료진 계정 (Django 기본)
   - `Doctor`: 의사 정보
   - `Department`: 진료과 정보

3. **OCS (처방전달시스템)**
   - `Order`: 주문 정보 (처방/검사/영상)
   - `OrderItem`: 주문 항목 상세
   - `DrugInteraction`: 약물 상호작용 정보

4. **의료 이미지**
   - `MedicalImage`: 의료 이미지 메타데이터
   - `ImageAnalysis`: AI 분석 결과

5. **진료 기록**
   - `MedicalRecord`: 진료 기록
   - `Prescription`: 처방전

### 슬라이드 17: 검사 프로세스 (맘모그래피, 병리, MRI)
**제목**: 검사 프로세스

**1. 맘모그래피 (Mammography)**
- **목적**: 유방암 조기 발견
- **프로세스**:
  ```
  의사 → 맘모그래피 촬영 의뢰 (OCS)
    ↓
  방사선과 → 4장 촬영 (L-CC, R-CC, L-MLO, R-MLO)
    ↓
  방사선과 → DICOM 업로드 (Orthanc)
    ↓
  영상의학과 → AI 분석 실행
    ↓
  AI 분석 결과 (Mass, Calcification, Asymmetry, Normal)
    ↓
  판독서 작성
  ```

**2. 병리 이미지 (Pathology)**
- **목적**: 조직 검사, 종양 여부 판별
- **프로세스**:
  ```
  의사 → 조직 검사 의뢰
    ↓
  병리과 → SVS 파일 업로드
    ↓
  SVS → DICOM 변환 (Orthanc)
    ↓
  영상의학과 → AI 분석 실행 (CLAM 모델)
    ↓
  AI 분석 결과 (Normal/Tumor)
    ↓
  판독서 작성
  ```

**3. MRI (Magnetic Resonance Imaging)**
- **목적**: 3D 영상 분석, 종양 세그멘테이션
- **프로세스**:
  ```
  의사 → MRI 촬영 의뢰
    ↓
  방사선과 → 4-ch DCE-MRI 촬영
    ↓
  방사선과 → DICOM 업로드 (Orthanc)
    ↓
  영상의학과 → AI 세그멘테이션 실행 (SwinUNETR)
    ↓
  세그멘테이션 결과 (DICOM SEG)
    ↓
  판독서 작성
  ```

**시각 자료**:
- 각 검사 프로세스 플로우차트
- 검사 이미지 예시

- **모델**: ResNet50
- **입력**: 4장 맘모그래피 이미지 (L-CC, R-CC, L-MLO, R-MLO)

```

6. 각 이미지별 분류 결과 반환
   ↓
7. 프론트엔드에 결과 표시
```

**결과 표시**:
- 각 뷰별 예측 클래스

## Part 6: 핵심 시스템 (OCS/RIS/LIS)

### 슬라이드 35: OCS (처방전달시스템) 개요
**제목**: OCS (Order Communication System)

**OCS란?**
- 의사가 처방전, 검사, 영상 촬영을 주문하고 각 부서로 자동 전달하는 시스템

**주문 유형**:
1. **처방전** (Prescription)
   - 약국으로 전달
   - 약물 상호작용 체크
   - 알레르기 체크

2. **검사** (Lab Test)
   - 검사실로 전달
   - LIS 연동

3. **영상 촬영** (Imaging)
   - 방사선과로 전달
   - RIS 연동

4. **조직 검사** (Tissue Exam)
   - 병리과로 전달

**주문 상태**:
- 대기중 (pending) → 전달됨 (sent) → 처리중 (processing) → 완료 (completed)

**시각 자료**:
- OCS 시스템 구조도
- 주문 프로세스 플로우차트

---

**제목**: OCS 주요 기능

**1. 주문 생성 및 관리**

**2. 자동 검증 시스템**
- **약물 상호작용 체크**
  - 두 약물 간 상호작용 검사
  - 심각도 분류 (경미/중등도/심각)
  - 심각한 상호작용 발견 시 주문 전달 불가

- **알레르기 체크**

**3. 우선순위 관리**

**4. 실시간 알림**
- 주문 상태 변경 시 알림
- 부서별 알림

**시각 자료**:
